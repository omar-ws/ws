Solução
500 Erro interno do servidor significa que o servidor encontrou uma condição inesperada ou um problema de configuração que o impede de atender à solicitação. É uma mensagem de erro muito genérica que exige informações dos registros disponíveis para rastreá-la.

Vamos começar investigando ativamente quaisquer pistas sobre a instância do Amazon Lightsail que nos ajudem a definir a causa raiz. Vamos examinar os registros de erros do Apache.

Etapa 1: siga o arquivo de log localizado em /opt/bitnami/apache/logs/error_log, isso nos dará uma dica do que está acontecendo. Execute o comando abaixo na janela ativa do navegador SSH.

Comando:

sudo tail /opt/bitnami/apache/logs/error_log
você deve ver algo semelhante às linhas abaixo que são relatadas nos registros

Saída:

[Wed Jul 00 00:00:00.787780 0000][proxy_fcgi:error] [pid 3471:tid 139651851409088] [client 0.0.0.0:32194] AH01071: Got error 'PHP message: PHP Warning:  require(/opt/bitnami/wordpress/wp-includes/version.php): Failed to open stream: Permission denied in /opt/bitnami/wordpress/wp-settings.php on line 33; PHP message: PHP Fatal error:  Uncaught Error: Failed opening required '/opt/bitnami/wordpress/wp-includes/version.php' (include_path='.:/opt/bitnami/php/lib/php') in /opt/bitnami/wordpress/wp-settings.php:33\nStack trace:\n#0 /bitnami/wordpress/wp-config.php(111): require_once()\n#1 /opt/bitnami/wordpress/wp-load.php(50): require_once('...')\n#2 /opt/bitnami/wordpress/wp-blog-header.php(13): require_once('...')\n#3 /opt/bitnami/wordpress/index.php(17): require('...')\n#4 {main}\n  thrown in /opt/bitnami/wordpress/wp-settings.php on line 33', referer: https://0.0.0.0/wp-admin/
O código de erro AH01071 é gerado pelo Apache quando os scripts do lado do servidor não conseguem executar uma operação e saem abruptamente com um código de erro fatal.
De acordo com o erro acima, o script /opt/bitnami/wordpress/wp-settings.php não conseguiu abrir/incluir um script PHP localizado em /opt/bitnami/wordpress/wp-includes/version.php devido a um erro de Permissão negada.

Etapa 2: Examine as permissões do diretório definidas no /opt/bitnami/wordpress/wp-includes.

Comando:

ls -l /opt/bitnami/wordpress | grep wp-includes
Saída:

d--------- 30 bitnami daemon 12288 Jun  7 13:42 wp-includes
As permissões do diretório são definidas como 000, o que significa que nenhum usuário ou grupo seria capaz de ler, gravar ou executar nele, pois as tentativas de ler qualquer arquivo de /opt/bitnami/wordpress/wp-includes/ simplesmente falhariam.

Para remediar o problema 500, devemos fornecer permissões suficientes para que o daemon do grupo leia e grave arquivos. Execute os comandos abaixo para modificar as permissões e reiniciar o apache

Comando (s):

sudo chmod 775 /opt/bitnami/wordpress/wp-includes
sudo /opt/bitnami/ctlscript.sh restart apache
